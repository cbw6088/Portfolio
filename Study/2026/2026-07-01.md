## 1. 메모리 교체 (페이지 교체 알고리즘) ★★★★

> 3개 프레임, 페이지 참조 순서: **2, 3, 2, 1, 5, 2, 4** — 기출에서 프레임 수·참조 순서만 바뀌어 출제

### 1.1 FIFO (First In First Out)

- **원리**: 가장 먼저 들어온 페이지를 가장 먼저보냄
- **풀이 방법**
  1. 밀어내기 식으로 풀기
  2. 이전에 있던 페이지 재참조 시 순서 유지
  3. 메모리에 페이지가 있는지 확인하며 Page Fault 체크
- **답**: Page Fault **6번**

### 1.2 LRU (Least Recently Used)

- **원리**: 가장 오랫동안 참조되지 않은 페이지를보냄
- **풀이 방법**
  1. 밀어내기 식으로 풀기
  2. 이전에 있던 페이지 재참조 시 **순서 위로** 이동
  3. 메모리에 페이지가 있는지 확인하며 Page Fault 체크
- **답**: Page Fault **5번**

### 1.3 LFU (Least Frequently Used)

- **원리**: 참조 횟수가 가장 적은 페이지를보냄
- **동률 시**: FIFO 알고리즘으로 교체
- **풀이 방법**
  1. 참조 횟수 적기
  2. 밀어내기 식으로 풀기 (참조 횟수 고려)
  3. 재참조 시 순서 유지 (동률 시 FIFO 조건 적용)
  4. 메모리에 페이지가 있는지 확인하며 Page Fault 체크
- **답**: Page Fault **5번**

| 알고리즘 | 교체 기준 | FIFO 예시(3프레임) |
|----------|-----------|-------------------|
| FIFO | 가장 먼저 들어온 페이지 | 6회 |
| LRU | 가장 오래 참조 안 된 페이지 | 5회 |
| LFU | 참조 횟수 가장 적은 페이지 | 5회 |

## 2. 프로세스 스케줄링 ★★

### 2.1 선점형 vs 비선점형

| 구분 | 설명 |
|------|------|
| 비선점형 | CPU 할당받으면 종료까지 유지 |
| 선점형 | 실행 중인 프로세스를 중단하고 다른 프로세스에 강제 할당 가능 |

### 2.2 스케줄링 알고리즘

| 비선점형 | 설명 | 선점형 | 설명 |
|----------|------|--------|------|
| FIFO | 도착 순서대로 할당 | Round Robin (RR) | FIFO + 시간 할당량 |
| SJF | 실행 시간 짧은 프로세스 우선 | SRT | 선점형 SJF |
| HRN | SJF + 대기 시간 고려 | | |
| 기한부 | 마감시간 임박한 프로세스 우선 | | |
| 우선순위 | 우선순위에 따라 할당 (비선점·선점 둘 다 가능) | | |

### 2.3 기출 예제 — 공통 프로세스

| 프로세스 | 도착 시간 | 실행 시간 |
|----------|-----------|-----------|
| P1 | 0 | 3 |
| P2 | 1 | 1 |
| P3 | 2 | 4 |
| P4 | 3 (또는 4) | 2 |

**비선점형 SJF** (P4 도착 3)

- 평균 대기 시간: (0 + 2 + 4 + 1) / 4 = **1.75**
- 평균 반환 시간: (3 + 3 + 8 + 3) / 4 = **4.25**
- 반환 시간 = 대기 시간 + 실행 시간

**SRT** (선점형 SJF)

- 평균 대기 시간: (1 + 0 + 4 + 1) / 4 = **1.5**
- 평균 반환 시간: (4 + 1 + 8 + 3) / 4 = **4**

**Round Robin** (시간 할당량 = 2, P4 도착 4)

- 평균 대기 시간: (3 + 1 + 4 + 2) / 4 = **2.5**
- 평균 반환 시간: (6 + 2 + 8 + 4) / 4 = **5**

> RR 풀이: 대기실(큐) 1번부터 실행 / O=도착, V=실행, X=대기

## 4. 라우팅 프로토콜 ★★★★

| 구분 | 설명 |
|------|------|
| 정적 라우팅 | 고정된 경로 사용 |
| 동적 라우팅 | 경로 계산, 동적 경로 변경 |

**AS (Autonomous System)**: 자율시스템 — 네트워크 영역

| 분류 | 프로토콜 | 알고리즘 | 키워드 |
|------|----------|----------|--------|
| 내부 (IGP) | RIP | 거리 벡터 (벨만-포드) | 홉 수 제한, 초기 전체 라우팅 테이블 교환 |
| 내부 (IGP) | OSPF | 링크 상태 (다익스트라) | 큰 네트워크, AS를 지역으로 분할 |
| 외부 (EGP) | BGP | 경로 벡터 | 서로 다른 AS 간, 변경사항만 교환 |

> RIP: **홉 수** 기준 최단 경로 / OSPF: **경로 비용** 기준 최단 경로

## 5. 네트워크 계층 프로토콜 ★★★

> 역할: 데이터 전송 최적 경로 제공

| 프로토콜 | 핵심 키워드 |
|----------|-------------|

| RIP / OSPF / BGP | 라우팅 프로토콜 (위 참고) |

## 6. 데이터 링크 계층 프로토콜 ★★

> 역할: 노드 사이 데이터 전송 및 오류 수정

| 프로토콜 | 핵심 키워드 |
|----------|-------------|
| HDLC | 비트 지향, 데이터 흐름 제어·오류 보정, 비트 열 삽입 |
| ATM | 비동기식 시분할 다중화, 패킷형 전송 |
| PPP | 두 노드 직접 연결 시 사용 |
| 프레임 릴레이 | 중계/다중화만 수행, 고속 데이터 전송 |

### 6.1 HDLC 프레임

| 프레임 | 제어부 시작 | 역할 |
|--------|-------------|------|
| I (정보) | 0 | 데이터 전달 |
| S (감시/감독) | 10 | 오류·흐름 제어 |
| U (비번호) | 11 | 링크 동작모드 설정 |

### 6.2 HDLC 동작 모드

| 동작 모드 | 키워드 |
|-----------|--------|
| NRM (정규 응답) | 보조국은 폴 메시지 수신 시에만 송신 |
| ARM (비동기 응답) | 포인트 투 포인트 불균형 링크, 종국이 허가 없이 송신 |
| ABM (비동기 균형) | 포인트 투 포인트 균형 링크, 혼합국끼리 허가 없이 전송 |

### 6.3 데이터 링크 오류 제어

| 구분 | 종류 | 키워드 |
|------|------|--------|
| FEC (오류 자체 수정) | 해밍 코드 | 1비트 수정 가능 |
| FEC | 상승 코드 | 여러 비트 수정 가능 |
| BEC (재전송 요구) | 패리티 검사 | 패리티 비트 1비트 추가 |
| BEC | CRC | 다항식 기반 오류 검사 |
| BEC | 블록합 검사 | 2차원 패리티 검사 |
| ARQ | 자동반복 요청 | 재전송 기반 오류 제어 |

> 기출 키워드: **해밍코드, 패리티검사, CRC**

## 9. 서브넷 마스크 ★★★

### 9.1 IP 주소 구하는 방법

| 주소 | 구하는 방법 |
|------|-------------|
| 네트워크 주소 | 호스트 주소 **AND** 서브넷 마스크 |
| 호스트 주소 | 전체 IP 중 네트워크·브로드캐스트 주소 제외 |
| 브로드캐스트 주소 | 네트워크 주소 **OR** (NOT 서브넷 마스크) |

### 9.2 사용 가능한 IP 주소 개수

```
서브넷 마스크 255.255.255.192
→ 2진수: 11111111.11111111.11111111.11000000
→ 뒤의 0 개수 = 호스트 비트 = 6개
→ 2^6 = 64개 (네트워크·브로드캐스트 제외 시 62개 사용 가능)
```

## 10. 서브넷팅 ★★★

### 10.1 CIDR 표기법

```
네트워크 주소 + 서브넷 마스크 = CIDR

192.168.1.0 + 255.255.255.0 (1이 24개) = 192.168.1.0/24
```

### 10.2 FLSM (Fixed Length Subnet Mask)

> 234.122.1.0/24를 FLSM으로 3개 서브넷 분할 → 2^n ≥ 3 → **n = 2**

```
234.122.1.0 → 11101010.01111010.00000001.00000000

00XXXXXX → 1번 서브넷
01XXXXXX → 2번 서브넷
10XXXXXX → 3번 서브넷

2번 서브넷 브로드캐스트:
11101010.01111010.00000001.01111111 → 234.122.1.127
```

## 11. Shell Script ★★★

### 11.1 기본 유닉스(UNIX) 명령어

| 명령어 | 키워드 | 명령어 | 키워드 |
|--------|--------|--------|--------|
| pwd | 현재 경로 확인 | grep | 파일에서 특정 패턴 검색 |
| ls | 디렉토리 내용 출력 | find | 파일 검색 |
| cd | 디렉토리 이동 | chmod | 접근 권한 변경 |
| cp | 파일·디렉토리 복사 | ps | 실행 중인 프로세스 목록 |
| mv | 이동 또는 이름 변경 | kill | 프로세스 종료 |
| rm | 파일·디렉토리 삭제 | clear | 터미널 화면 지우기 |
| mkdir | 새 디렉토리 생성 | echo | 문자열 출력 |
| cat | 파일 내용 출력·연결 | touch | 빈 파일 생성·수정 시간 변경 |

### 11.2 chmod — 접근 권한 변경

```bash
chmod [사용자권한][그룹권한][기타권한] [파일명]
# 예: chmod 7 5 4 file.txt
```

| 권한 | 기호 | 8진수 | 대상 | 기호 |
|------|------|-------|------|------|
| 읽기 | r | 4 | 사용자 (u) | 소유자 |
| 쓰기 | w | 2 | 그룹 (g) | 파일이 속한 그룹 |
| 실행 | x | 1 | 기타 (o) | 그 외 모든 사용자 |

```
chmod 7 5 4
  7 = 4+2+1 (읽기+쓰기+실행)
  5 = 4+1   (읽기+실행)
  4 = 4     (읽기)
```

## 12. 시험 포인트 체크리스트

- [ ] 페이지 교체 — FIFO(6) / LRU(5) / LFU(5), 3프레임·참조 2,3,2,1,5,2,4
- [ ] LFU 동률 시 FIFO로 교체
- [ ] 비선점 SJF vs SRT vs RR — 평균 대기·반환 시간 계산
- [ ] 반환 시간 = 대기 시간 + 실행 시간
- [ ] 데이터그램(비연결) vs 가상회선(연결형)
- [ ] RIP(거리벡터·홉수) vs OSPF(링크상태·비용) vs BGP(경로벡터·AS간)
- [ ] ARP(IP→MAC) vs RARP(MAC→IP)
- [ ] ICMP — Ping of Death / NAT — 사설→공인 IP
- [ ] HDLC — I(0) / S(10) / U(11) 프레임, NRM·ARM·ABM
- [ ] 오류제어 — FEC(해밍·상승) vs BEC(패리티·CRC)
- [ ] 포트번호 — SSH(22), Telnet(23), HTTP(80), HTTPS(443), FTP(21), SMTP(25), DNS(53)
- [ ] 네트워크 주소 = IP AND 마스크 / 브로드캐스트 = 네트워크 OR NOT 마스크
- [ ] 사용 가능 IP = 2^(호스트 비트) − 2 (네트워크·브로드캐스트 제외)
- [ ] CIDR — 마스크의 1 개수 = /숫자
- [ ] FLSM — 2^n ≥ 서브넷 수, n비트 차지
- [ ] chmod 8진수 — r=4, w=2, x=1

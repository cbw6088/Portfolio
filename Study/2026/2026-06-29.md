## 1. SQL 명령어 ★★★★★

### 1.1 DML / DDL / DCL — 상황별 쿼리

| 상황 | 쿼리 |
|------|------|
| 데이터 추가 | `INSERT INTO 테이블(컬럼1, 컬럼2...) VALUES (값1, 값2...);` |
| 데이터 복제 | `INSERT INTO 테이블1(컬럼1...) SELECT 컬럼3, 컬럼4... FROM 테이블2;` |
| 데이터 삭제 | `DELETE FROM 테이블 WHERE 조건;` |
| 데이터 수정 | `UPDATE 테이블 SET 컬럼1=값1, 컬럼2=값2 WHERE 조건;` |
| 테이블 제거 | `DROP TABLE 테이블 RESTRICT;` (참조 시 삭제 불가) / `CASCADE;` (참조 테이블까지 삭제) |
| 권한 부여 | `GRANT 권한 ON 테이블 TO 사용자;` (SELECT, INSERT 등) |
| 권한 회수 | `REVOKE 권한 ON 테이블 FROM 사용자;` |
| 뷰 생성 | `CREATE VIEW 뷰이름 AS SELECT ... FROM 테이블 WHERE 조건;` |
| 뷰 제거 | `DROP VIEW 뷰이름 RESTRICT;` / `CASCADE;` |
| 컬럼 추가 | `ALTER TABLE 테이블 ADD 컬럼이름 데이터타입;` |
| 컬럼 제거 | `ALTER TABLE 테이블 DROP COLUMN 컬럼이름;` |
| 인덱스 생성 | `CREATE INDEX 인덱스이름 ON 테이블(컬럼1, 컬럼2...);` |

### 1.2 SELECT — 정렬·조건·집계

| 상황 | 쿼리 |
|------|------|
| 컬럼 기준 정렬 | `ORDER BY 컬럼1 ASC` (오름차순, 생략 가능) / `DESC` (내림차순) |
| 두 컬럼 정렬 | `ORDER BY 컬럼1 DESC, 컬럼2 DESC` — 앞 컬럼 우선 정렬 |
| 범위 조회 | `WHERE 컬럼1 BETWEEN 1 AND 3` / `WHERE 컬럼1 IN (1,2,3)` |
| 특정 문자로 시작 | `WHERE 컬럼1 LIKE '특정문자%'` |
| 특정 문자 포함 | `WHERE 컬럼1 LIKE '%특정문자%'` |
| 앞에 한 글자만 | `WHERE 컬럼1 LIKE '_특정문자'` |
| NULL 조회 | `WHERE 컬럼1 IS NULL` |
| 별칭 | `SELECT 컬럼1 AS 별칭1` — `AS` 생략 가능 |
| 합계에 조건 | `SELECT 컬럼2, SUM(컬럼1) AS 합계 FROM 테이블 WHERE 조건 GROUP BY 컬럼2 HAVING SUM(컬럼1) 조건;` |
| 개수 세기 | `COUNT(컬럼명)` — NULL 아닌 행 / `COUNT(*)` — 모든 행 |

### 1.3 집계 함수

| 함수 | 역할 |
|------|------|
| `COUNT(*, 컬럼명)` | `*` = 모든 행 / 컬럼명 = NULL 아닌 행 |
| `SUM(컬럼명)` | 합계 |
| `AVG(컬럼명)` | 평균 |
| `MAX(컬럼명)` | 최대값 |
| `MIN(컬럼명)` | 최소값 |

### 1.4 서브쿼리

| 연산자 | 의미 |
|--------|------|
| `IN` | 서브쿼리 값 중 **하나라도** 일치하면 반환 |
| `> ALL` | 서브쿼리 **모든** 값보다 커야 반환 |
| `> ANY` (= `SOME`) | 서브쿼리 값 **하나라도**보다 크면 반환 |
| `EXISTS` | 서브쿼리 결과가 **존재**하면 반환 |

**중첩 서브쿼리 풀이 순서**: 가장 안쪽 → 바깥쪽 → 메인 쿼리

### 1.5 집합 연산자

| 연산자 | 설명 | 쿼리 예시 |
|--------|------|-----------|
| `UNION` | 중복 제거 합집합 | `SELECT ... FROM 테이블1 UNION SELECT ... FROM 테이블2` |
| `UNION ALL` | 중복 포함 합집합 | 위와 동일, `UNION ALL` |
| `INTERSECT` | 교집합 | `... INTERSECT SELECT ...` |
| `EXCEPT` | 차집합 (Oracle: `MINUS`) | `... EXCEPT SELECT ...` |

### 1.6 쿼리 해석 — 자주 나오는 패턴

```sql
-- AND가 OR보다 우선 → (조건1 AND 조건2) OR 조건3
SELECT COUNT(*) FROM 테이블 WHERE 조건1 AND 조건2 OR 조건3;

-- 중복 제외한 고유 값 개수
SELECT COUNT(DISTINCT 컬럼1) FROM 테이블;
```

## 2. 조인(JOIN) ★★★★★

| 종류 | 키워드 | 설명 |
|------|--------|------|
| **세타 조인** | `(INNER) JOIN` + 비등호 조건 | 조건 만족 튜플만 반환 |
| **동등 조인** | `(INNER) JOIN` + `=` | 세타 조인 중 등호 일치 |
| **자연 조인** | `NATURAL JOIN` | 동등 조인 + 중복 속성 제거 |
| **외부 조인** | `LEFT/RIGHT/FULL OUTER JOIN` | 불일치 튜플도 반환, NULL 채움 |
| **교차 조인** | `CROSS JOIN` | 모든 튜플 조합 (카티션) |

```sql
-- 세타 조인
SELECT T1.컬럼1, T2.컬럼2 FROM 테이블1
INNER JOIN 테이블2 ON T1.컬럼1 > T2.컬럼1;

-- 동등 조인 (INNER 생략 가능)
SELECT T1.컬럼1, T2.컬럼2 FROM 테이블1
INNER JOIN 테이블2 ON T1.컬럼1 = T2.컬럼1;

-- 자연 조인
SELECT 컬럼1, 컬럼2 FROM 테이블1 NATURAL JOIN 테이블2;

-- 외부 조인
SELECT T1.컬럼1, T2.컬럼2 FROM 테이블1
LEFT OUTER JOIN 테이블2 ON T1.컬럼1 = T2.컬럼1;

-- 교차 조인 (= FROM 테이블1, 테이블2)
SELECT T1.컬럼1, T2.컬럼2 FROM 테이블1 CROSS JOIN 테이블2;
```

## 3. 트랜잭션 ★★★★

### 3.1 ACID

| 특성 | 키워드 |
|------|--------|
| **원자성** (Atomicity) | All or Nothing — 모두 실행되거나 하나도 실행 안 됨 |
| **일관성** (Consistency) | 실행 후에도 DB 일관성 보장 |
| **고립성** (Isolation) | 다른 트랜잭션 끼어들 수 없음, 동시 실행 제어 |
| **지속성** (Durability) | 성공 완료 결과 = 영구 저장 |

### 3.2 동시성 제어 — Locking

| 락 | 용도 |
|----|------|
| **S-Lock** (공유 락) | 읽기 — 여러 트랜잭션 동시 읽기 가능, **쓰기 불가** |
| **X-Lock** (배타 락) | 쓰기 — 다른 트랜잭션 **읽기/쓰기 모두 불가** |

### 3.3 회복 기법

**핵심 요소**

| 요소 | 키워드 |
|------|--------|
| **Redo** (재실행) | 장애 전 DB로 복구 — Start/Commit 기록 필요 |
| **Undo** (취소) | 작업 모두 취소 — Start 있지만 Commit 없는 경우 |

**기법 종류**

| 기법 | 키워드 |
|------|--------|
| **지연 갱신** | 트랜잭션 완료까지 DB 변경 지연 → **Redo만** 사용 |
| **즉각 갱신** | 즉시 DB 반영 → **Redo + Undo** 모두 필요 |
| **체크포인트 회복** | 특정 시점 DB 상태 저장, 장애 시 체크포인트로 회복 |
| **그림자 페이징** | 원본 복사(그림자 페이지) 생성, 실패 시 그림자 페이지로 대체 |

### 3.4 스키마 3단계

| 스키마 | 키워드 |
|--------|--------|
| **외부 스키마** | 사용자/개발자 관점, 사용자 뷰, 서브스키마 |
| **개념 스키마** | 전체 뷰, 개체 간 관계·제약 조건 정의 |
| **내부 스키마** | 물리 저장장치 관점, 실제 레코드 형식 정의 |

## 4. 관계대수 / 관계해석 ★★

| 구분 | 관계대수 | 관계해석 |
|------|----------|----------|
| 성격 | **절차적** 언어 | **비절차적** 언어 (Codd) |
| 관점 | **어떻게**(HOW) 가져올지 명시 | **무엇을**(WHAT) 원하는지 명시 |

### 4.1 일반 집합 연산자

| 연산자 | 기호 | 설명 |
|--------|------|------|
| 합집합 | ∪ | 중복 제외, 두 릴레이션 튜플 모두 반환 |
| 교집합 | ∩ | 공통 튜플 반환 |
| 차집합 | − | 왼쪽에만 있고 오른쪽에 없는 튜플 |
| 카티션 프로덕트 | × | 모든 순서쌍 조합 |

### 4.2 순수 관계 연산자

| 연산자 | 기호 | 설명 |
|--------|------|------|
| 셀렉트 (Select) | σ | 조건 만족 **튜플** 반환 (수평) |
| 프로젝트 (Project) | π | 조건 만족 **열** 반환 (수직) |
| 조인 (Join) | ⨝ | 공통 속성 기준 결합 |
| 디비전 (Division) | ÷ | A÷B : B의 모든 튜플에 대해 관계 맺는 A의 튜플 반환 |

> **주어진 릴레이션에 대해 연산자 수행 결과를 직접 쓸 수 있어야 함** (예: R ÷ S)

## 5. 관계 데이터 모델 ★★

| 구성요소 | 키워드 |
|----------|--------|
| **Attribute** (속성) | 릴레이션의 각 열(Column), 데이터 필드 |
| **Tuple** (튜플) = Record | 릴레이션의 각 행(Row), 중복값 포함 가능 |
| **Cardinality** (카디널리티) | 튜플의 수, 고유 값의 총 개수 |
| **Degree** (차수) | 릴레이션에 포함된 속성의 수 |
| **Domain** (도메인) | 하나의 속성이 가질 수 있는 값들의 집합 |
| **Relation Instance** = Extension | 특정 시점의 실제 데이터 집합 |
| **Relation** (릴레이션) | 행·열로 구성된 2차원 테이블 자체 |
| **Relation Schema** | DB 구조/제약 조건 명세 (이름, 속성, 타입) |
| **Identifier** (식별자) | 각 튜플을 고유하게 식별하는 속성 (예: 후보키) |

## 6. 요개논물 (DB 설계) ★★★★

### 6.1 데이터 모델 3가지 구성요소

| 구성 요소 | 핵심 개념 | 설명 |
|-----------|-----------|------|

### 6.2 DB 설계 4단계

| 단계 | 핵심 활동 | 주요 산출물 |
|------|-----------|-------------|
| **요구조건 분석** | 사용자 요구사항 수집·분석 | 요구조건 명세서 |

**전체 프로세스**: 요구조건 분석 → 개념적 → 논리적 → 물리적 → **구현**

## 7. 정규화 ★★★

### 7.1 정규형

| 정규형 | 조건 |
|--------|------|
| **제1정규형** | 모든 속성 값 = **원자값** (Atomic value) |
| **제2정규형** | **부분 함수 종속** 제거 (완전 함수 종속 성립) |
| **제3정규형** | **이행 함수 종속** 제거 |
| **BCNF** | 모든 **결정자가 후보키** |
| 제4정규형 | **다치 종속** 제거 |
| 제5정규형 | **조인 종속** 제거 |

### 7.2 함수 종속

| 종류 | 설명 |
|------|------|
| **완전 함수 종속** | 기본키 **전체**에 대해서만 종속 |
| **부분 함수 종속** | 기본키의 **일부** 속성에 종속 |
| **이행 함수 종속** | A→B, B→C, A→C 성립 (A=후보키, B·C=후보키에 미포함 속성) |

- **A→B**: 속성 A의 값이 속성 B의 값을 결정
- **반정규화**: 정규화된 구조를 의도적으로 통합·분리·중복 허용 → **성능 최적화**

## 8. 키(Key) ★★★★

> **기대(후)슈**: 기본키 + 대체키 = **후보키** ⊂ **슈퍼키**

| 키 종류 | 키워드 |
|---------|--------|
| **기본키** | 후보키 중 선택, **NOT NULL** |
| **대체키** | 후보키 중 선택되지 못한 나머지 |
| **후보키** | 유일성 O, 최소성 O — 튜플 구별 기준 |
| **슈퍼키** | 유일성 O, 최소성 X — 튜플 유일 식별 |
| **외래키** | 한 릴레이션 컬럼이 다른 릴레이션의 **기본키** 참조 |

## 9. 무결성 ★★

| 무결성 | SQL 구현 | 키워드 |
|--------|----------|--------|
| **개체** | `PRIMARY KEY` | 기본키 = NOT NULL + 유일 |
| **참조** | `FOREIGN KEY` | 외래키 = 참조 테이블 기본키값 or NULL |

| **사용자** | `TRIGGER` | 비즈니스 규칙에 맞게 직접 정의 |
| **키** | `UNIQUE` | 릴레이션에 적어도 하나의 키 존재 |

## 10. 스토리지 (RAID) ★★★

| RAID | 핵심 | 키워드 |
|------|------|--------|
| **RAID 0** | 분산 저장 | 스트라이핑(Striping), 패리티·복제 없음 |
| **RAID 1** | 복제 저장 | 미러링(Mirroring), 높은 안정성 |
| **RAID 5** | 스트라이핑 + 분산 패리티 | 패리티로 오류 검출·복구, 최소 **3개** 디스크 |
| **RAID 6** | 스트라이핑 + 이중 분산 패리티 | 독립적 패리티 2개, 최소 **4개** 디스크 |
| **RAID 10** | 미러링 후 스트라이핑 | RAID 1 + RAID 0, 최소 **4개** 디스크, 성능·안정성 모두 높음 |

## 12. 시험 포인트 체크리스트

- [ ] `DROP TABLE/VIEW` — **RESTRICT** vs **CASCADE** 차이

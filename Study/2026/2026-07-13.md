# 정보처리기사 실기 — 서브넷·브로드캐스트 수신 IP

---

## 0. 한눈에 보기

| 항목 | 내용 |
|------|------|
| 출제 포인트 | IP + 서브넷 마스크 → **같은 네트워크(브로드캐스트 도메인)** 판별 |
| 핵심 흐름 | 서브넷 마스크로 **네트워크 주소·범위** 구하기 → 보기 IP가 범위 안인지 확인 |
| 자주 틀리는 점 | 3번째 옥텟만 보고 판단 / 브로드캐스트·네트워크 주소를 호스트로 포함 |

> **한 줄 요약**: 브로드캐스트는 **같은 서브넷 안의 호스트**만 수신한다. 먼저 **네트워크 범위**를 구한다.

---

## 1. 문제

IP 주소가 **192.168.35.10**, 서브넷 마스크가 **255.255.252.0**인 PC에서 **브로드캐스팅**으로 다른 IP에 정보를 전달한다고 할 때, 수신할 수 있는 알맞은 IP를 **보기에서 골라 모두** 작성하시오.

**[보기]**

| 기호 | IP 주소 |
|------|---------|
| ㄱ | 192.168.34.1 |
| ㄴ | 192.168.32.19 |
| ㄷ | 192.168.35.200 |
| ㄹ | 192.168.33.138 |
| ㅁ | 192.168.35.50 |

**정답**: **ㄱ, ㄴ, ㄷ, ㄹ, ㅁ** (전부)

---

## 2. 핵심 개념

### 2.1 용어 정리

| 용어 | 의미 |
|------|------|
| 서브넷 마스크 | IP에서 **네트워크 부분**과 **호스트 부분**을 구분하는 값 |
| 네트워크 주소 | 해당 서브넷의 **시작 주소** (호스트 비트가 모두 0) |
| 브로드캐스트 주소 | 해당 서브넷의 **끝 주소** (호스트 비트가 모두 1) |
| 브로드캐스트 | 같은 서브넷의 **모든 호스트**에게 한 번에 보내는 통신 |
| 브로드캐스트 도메인 | 브로드캐스트가 **도달할 수 있는 범위** (= 같은 서브넷) |

### 2.2 브로드캐스트 수신 규칙

- PC가 보낸 **브로드캐스트 패킷**은 **같은 서브넷에 속한 호스트**만 수신한다.
- 다른 서브넷에 있는 IP는 **라우터를 거치지 않는 한** 수신하지 못한다.
- 따라서 **보기 IP가 PC와 같은 네트워크 범위에 있는지** 확인하면 된다.

### 2.3 서브넷 마스크 255.255.252.0 읽는 법

| 옥텟 | 값 | 이진수 | 의미 |
|------|-----|--------|------|
| 1 | 255 | 11111111 | 네트워크 (고정) |
| 2 | 255 | 11111111 | 네트워크 (고정) |
| 3 | 252 | 11111100 | 앞 6비트 네트워크, 뒤 2비트 호스트 |
| 4 | 0 | 00000000 | 호스트 |

- CIDR 표기: **/22** (네트워크 비트 22개)
- 3번째 옥텟에서 **블록 크기** = 256 − 252 = **4**
- 3번째 옥텟은 **4씩 증가**: 0, 4, 8, 12, …, 28, **32**, **36**, …

---

## 3. 풀이 방법 (4단계)

### Step 1 — 블록 크기 구하기

```
블록 크기 = 256 − (서브넷 마스크에서 0이 아닌 마지막 옥텟을 256에서 뺀 값)
         = 256 − 252 = 4
```

### Step 2 — 네트워크 주소 구하기

호스트 IP **192.168.35.10**의 3번째 옥텟 `35`가 속한 블록을 찾는다.

```
… 28, 32, 36 …
      ↑ 35는 32~35 구간 → 네트워크 시작 = 32
```

**네트워크 주소**: `192.168.32.0`

> 3번째 옥텟을 **블록 크기(4)의 배수**로 내림하면 네트워크 주소의 3번째 옥텟이 된다.  
> 35 ÷ 4 = 8 … 3 → 8 × 4 = **32**

### Step 3 — 네트워크 범위·브로드캐스트 주소

| 항목 | 주소 |
|------|------|
| 네트워크 주소 | 192.168.32.0 |
| 사용 가능 호스트 | 192.168.32.1 ~ 192.168.35.254 |
| 브로드캐스트 주소 | 192.168.35.255 |

```
블록 크기 4 → 3번째 옥텟이 32, 33, 34, 35까지 한 네트워크
다음 네트워크는 192.168.36.0 부터 시작
```

### Step 4 — 보기 IP 판별

3번째 옥텟이 **32 ~ 35** 사이이고, 호스트 주소(네트워크·브로드캐스트 제외)이면 **수신 가능**.

| 기호 | IP | 3번째 옥텟 | 같은 서브넷? | 수신 |
|------|-----|-----------|-------------|------|
| ㄱ | 192.168.34.1 | 34 | O (32~35) | **O** |
| ㄴ | 192.168.32.19 | 32 | O | **O** |
| ㄷ | 192.168.35.200 | 35 | O | **O** |
| ㄹ | 192.168.33.138 | 33 | O | **O** |
| ㅁ | 192.168.35.50 | 35 | O | **O** |

**정답**: ㄱ, ㄴ, ㄷ, ㄹ, ㅁ

---

## 4. 이진수로 확인 (검산)

```
IP      192.168.  35 .  10  →  00100011 . 00001010
마스크  255.255. 252 .   0  →  11111100 . 00000000
                              ─────────────────────
네트워크 192.168.  32 .   0  →  001000 00 . 00000000
```

- 마스크 `252` = `11111100` → 3번째 옥텟의 **앞 6비트**만 네트워크
- `35` AND `252` = `32` → 네트워크 3번째 옥텟 = **32**

보기 ㄱ `34`: `34` AND `252` = `32` → 같은 네트워크  
보기 ㄹ `33`: `33` AND `252` = `32` → 같은 네트워크

---

## 5. 시험에서 자주 틀리는 포인트

| 실수 | 올바른 처리 |
|------|-------------|
| 3번째 옥텟 숫자만 비교 | **블록 크기**로 네트워크 구간을 먼저 구할 것 |
| 35와 가까운 IP만 고름 | 같은 블록은 **32~35 전체** (34, 33도 포함) |
| 192.168.36.x를 같은 네트워크로 봄 | 36은 **다음 블록** 시작 → 수신 불가 |
| 192.168.31.x를 같은 네트워크로 봄 | 31은 **이전 블록** (28~31) → 수신 불가 |
| 브로드캐스트 주소를 호스트로 셈 | `.0`(네트워크), `.255`(브로드캐스트)는 호스트 아님 |

### 함정 보기 예시 (이 문제에는 없음)

| IP | 판별 |
|----|------|
| 192.168.36.10 | 3번째 옥텟 36 → **다른 서브넷** (수신 X) |
| 192.168.31.50 | 3번째 옥텟 31 → **다른 서브넷** (수신 X) |
| 192.168.32.0 | **네트워크 주소** (호스트 아님) |
| 192.168.35.255 | **브로드캐스트 주소** (호스트 아님) |

---

## 6. 자주 나오는 서브넷 마스크 표

| 서브넷 마스크 | CIDR | 블록 크기 (4번째 옥텟) | 3번째 옥텟 블록 |
|---------------|------|------------------------|-----------------|
| 255.255.255.0 | /24 | 256 | 1씩 증가 |
| 255.255.254.0 | /23 | — | 2씩 증가 (0,2,4…) |
| 255.255.252.0 | /22 | — | **4씩 증가** (0,4,8…) |
| 255.255.248.0 | /21 | — | 8씩 증가 |
| 255.255.0.0 | /16 | — | 256 (B클래스) |

---

## 7. 빠른 풀이 템플릿

```
1) 마스크에서 블록 크기 구하기 (256 − 마지막 옥텟)
2) 호스트 IP가 속한 블록 시작값 구하기 (옥텟 ÷ 블록크기 → 내림)
3) 네트워크 ~ 브로드캐스트 범위 적기
4) 보기 IP가 범위 안인지 하나씩 확인
5) "모두 고르시오" → 범위 안이면 전부 정답 후보
```

---

## 8. 암기 문장

```
1) 브로드캐스트 = 같은 서브넷만 수신.
2) 마스크 252 → 블록 크기 4 → 3번째 옥텟 4칸이 한 네트워크.
3) 35는 32~35 블록 → 32.0 ~ 35.255.
4) 보기의 3번째 옥텟이 32~35면 같은 네트워크.
```
